⚠️ TROUBLESHOOTING
───────────────────────────────────────────────────────────────────────────

PROBLEM: Nothing shows up (blank hero section)

  Check 1: File paths
    → Open ngb.html location
    → Verify this structure exists:
      frontend/ngb.html  ← Your HTML file
      ../assets/videos/0620(1).mp4  ← Your video
      ../assets/images/home/  ← Your images folder
  
  Check 2: Browser console (F12)
    → Look for red error messages
    → Common error: "Failed to load resource: net::ERR_FILE_NOT_FOUND"
    → Fix: Verify file paths match your folder structure

  Check 3: Media files
    → Check assets/videos/0620(1).mp4 exists
    → Check assets/images/home/ contains 4 .jpg files

───────────────────────────────────────────────────────────────────────────

PROBLEM: Video doesn't play (but images do)

  This is NORMAL in some browsers due to autoplay policies.
  
  → Chrome/Edge: Should work fine
  → Firefox: Should work fine
  → Safari: Might block autoplay until user interacts with page
  → Mobile: Often blocks autoplay to save data
  
  Solution: Image carousel fallback activates automatically.
  This is intentional behavior — no fix needed!

───────────────────────────────────────────────────────────────────────────

PROBLEM: Text not readable over media

  Check 1: Overlay exists
    → Open browser DevTools (F12) → Elements tab
    → Look for <div class="hero__media-overlay"></div>
    → Should be inside .hero__media container
  
  Check 2: Overlay opacity
    → If overlay too light, open frontend/styles/ngb.css
    → Find line ~746: background-color: rgba(26, 26, 26, 0.35);
    → Change 0.35 to 0.5 (darker) or 0.6 (much darker)
    → Save and refresh browser

───────────────────────────────────────────────────────────────────────────

PROBLEM: Images flickering during carousel transitions

  This means images aren't preloaded.
  
  → Check browser console for "Failed to load resource" errors
  → Verify image file names match exactly:
    - download (1).jpg  ← Note the space and parentheses
    - image0.jpg
    - image3 (1).jpg
    - image4 (1).jpg
  
  → Windows is case-insensitive, but web servers aren't
  → Make sure capitalization matches exactly

───────────────────────────────────────────────────────────────────────────

PROBLEM: Console shows "Required elements not found"

  This means HTML structure is wrong.
  
  → Open frontend/ngb.html
  → Search for: <section id="hero"
  → Verify this structure:
  
    <section id="hero" class="section section--hero">
      <div class="hero__media" aria-hidden="true">
        <!-- JavaScript will inject video or image carousel here -->
      </div>
      <div class="hero__content container">
        <div class="hero__text">
          ...
        </div>
      </div>
    </section>
  
  → If structure looks different, something went wrong with HTML update
